# Инструкция по постоянному деплою (Next.js)

Этот проект является фронтендом на базе **Next.js**. Для того чтобы он работал постоянно и был доступен по вашему домену, следуйте этой инструкции.

## Рекомендуемый хостинг: Vercel (Бесплатно и быстро)

[Vercel](https://vercel.com) — это создатели Next.js, поэтому деплой там самый простой и эффективный.

### Шаги по деплою:

1. **Зарегистрируйтесь на Vercel:** Используйте свой аккаунт GitHub.
2. **Импортируйте проект:**
   - Нажмите "Add New" -> "Project".
   - Выберите ваш репозиторий `ansdef/cloudco-final`.
3. **Настройте переменные окружения (Environment Variables):**
   - В процессе импорта разверните раздел "Environment Variables".
   - Добавьте переменную:
     - **Key:** `NEXT_PUBLIC_API_URL`
     - **Value:** URL вашего бэкенда (например, `https://your-backend-api.com/api`).
4. **Нажмите "Deploy":** Vercel автоматически соберет проект и выдаст вам постоянный URL (например, `cloudco-final.vercel.app`).

---

## Альтернатива: Netlify или Railway

### Netlify:
1. Подключите GitHub.
2. Выберите репозиторий.
3. Build command: `npm run build`
4. Publish directory: `.next`
5. Добавьте `NEXT_PUBLIC_API_URL` в настройках.

### Railway:
1. Выберите "New Project" -> "Deploy from GitHub repo".
2. Railway сам определит Next.js.
3. В разделе "Variables" добавьте `NEXT_PUBLIC_API_URL`.

---

## Важные замечания:
- **Бэкенд:** Этот фронтенд ожидает работающий API. Убедитесь, что ваш бэкенд (из первого репозитория) также задеплоен (например, на Railway или Render).
- **Домен:** В настройках Vercel/Netlify вы можете бесплатно подключить свой собственный домен (например, `www.cloudco.ru`).
